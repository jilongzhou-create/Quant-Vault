# Isolated Macro - 叙事驱动/目标敞口范式 研发文档

## 1. 项目概述

`isolated_macro/` 是 `trading_agent/` 项目中完全独立的一级模块，用于研发**叙事驱动的宏观估值策略**。与主系统（技术面信号驱动、开关式仓位）完全隔离，采用**连续敞口滑行**范式。

核心假设：商品价格由宏观因子（持有成本锚 + 计价锚）决定长期均衡水平，偏离均衡时产生均值回归机会；但 Regime 切换会导致残差持续漂移，需要趋势代理来感知并修正。

## 4. 模型策略逻辑

### 4.1 第一层：宏观估值模型（ECM Logs Levels 回归）

**公式：**
```
ln(GoldPrice) = β0 + β1 × DFII10 + β2 × ln(DTWEXBGS) + ε
```

- **DFII10**：10 年期 TIPS 实际收益率（持有成本锚）。TIPS 上升 → 持有黄金的机会成本增加 → 金价承压。预期 β1 < 0。
- **DTWEXBGS**：贸易加权美元指数（计价锚）。美元走强 → 以美元计价的黄金变贵 → 需求下降 → 金价承压。预期 β2 < 0。

**IS 校准结果（2007-01-02 ~ 2019-12-31，3260 个观测）：**

| 系数 | 值 | 标准误 | t 统计量 | p 值 | 显著性 |
|------|-----|--------|---------|------|--------|
| Intercept (β0) | 7.9727 | 0.0316 | 252.12 | 0.000 | *** |
| b_DFII10 (β1) | -0.0688 | 0.0046 | -14.91 | 0.000 | *** |
| b_lnDXY (β2) | -0.6760 | 0.0290 | -23.30 | 0.000 | *** |

| 统计量 | 值 |
|--------|-----|
| R-squared | 0.8017 |
| Adj. R-squared | 0.8016 |
| F-statistic | 6579.19 |
| Prob(F) | 0.000 |
| **Durbin-Watson** | **0.024** |
| AIC | -18641.3 |
| BIC | -18617.3 |

**关键发现：** R²=0.80 说明模型解释力强，但 **DW=0.024** 表明残差存在极端正自相关——金价可以持续数年偏离理论价格而不回归。这是纯均值回归策略失败的根本原因。

### 4.2 第二层：估值偏差 → 目标敞口

**计算步骤：**

1. **FairValue**：将当日 TIPS 和 DXY 代入回归方程，还原为理论价格
   ```
   ln(FairValue) = 7.9727 + (-0.0688) × DFII10 + (-0.6760) × ln(DTWEXBGS)
   FairValue = exp(ln(FairValue))
   ```

2. **Valuation Spread**：对数偏差
   ```
   Spread = ln(MarketPrice) - ln(FairValue) = ln(MarketPrice / FairValue)
   ```

3. **Z-Score**：滚动 5 年（1260 日）标准化
   ```
   Z = (Spread - RollingMean_1260) / RollingStd_1260
   ```

4. **Raw Exposure**（纯均值回归信号）：
   ```
   RawExposure = -clip(Z / 2.0, -1, 1)
   ```
   Z > 0（高估）→ 做空；Z < 0（低估）→ 做多

### 4.3 第三层：趋势代理与敞口合成（Smart Trend）

引入 200 日 SMA 作为趋势代理（Satellite 模型），解决"好 Odds 但逆趋势"的问题。

**趋势信号计算：**
```
trend_dev = (Price - SMA200) / SMA200
trend_signal = clip(trend_dev / 0.15, -1, 1)
```

**六版敞口合成公式：**

| 版本 | 名称 | 公式 | 核心逻辑 |
|------|------|------|----------|
| V1 | 纯均值回归 | `RawExposure` | 只看估值，不看趋势 |
| V2 | 缩减敞口 | `Raw × ScalingScore` | 趋势冲突时缩减敞口至 0 |
| V2b | 趋势融合 | `Raw × Scale + Trend × (1-Scale)` | 冲突时跟随趋势 |
| V3 | 趋势主导 | `(1-w) × Raw + w × Trend` | 趋势强时跟趋势，弱时用估值 |
| V4 | Long-Only V3 | `clip(V3, 0, 1)` | V3 + 永不做空 |
| V5 | 纯趋势 | `trend_signal` | 完全不用估值 |

其中 V2/V2b 的 ScalingScore：
```
ScalingScore = clip(1 + sign(RawExposure) × trend_signal, 0, 1)
```

V3 的趋势权重：
```
trend_weight = clip(|trend_signal| / 0.3, 0, 1)
```

## 5. 回测引擎设计

### 5.1 MacroBacktestEngine 核心规则

| 铁律 | 实现 | 目的 |
|------|------|------|
| **shift(1)** | `position = target_exposure.shift(1)` | 昨天的目标敞口 × 今天的实际收益，杜绝 Look-ahead bias |
| **ffill** | `position = target_exposure.shift(1).ffill()` | 目标敞口优雅维持，不每天买卖 |
| **交易成本** | `cost = |Δposition| × 0.02%` | 单边手续费，惩罚频繁调仓 |

### 5.2 绩效指标

- 总收益、年化收益、年化波动率
- Sharpe Ratio（年化收益/年化波动）、Sharpe(rf=0)
- Calmar Ratio（年化收益/最大回撤）
- 最大回撤、回撤持续天数
- 胜率、盈亏比
- 换手率（Total Turnover）
- 偏度、峰度

## 7. 关键发现与诊断

### 7.1 为什么纯均值回归（V1）失败？

1. **DW=0.024**：残差极端自相关，Spread 一旦偏离会持续数年不回归
2. **结构性上行偏置**：黄金有天然的通胀对冲 + 避险属性，长期价格中枢上移
3. **叙事缺口**：TIPS + DXY 只解释了金融属性，遗漏了避险溢价、央行购金、QE 货币属性

### 7.2 为什么 V2/V2b 更差？

- **V2（缩减）**：趋势冲突时敞口归零，错过牛市收益，但做空端依然亏损
- **V2b（融合）**：趋势信号频繁翻转导致敞口来回切换，换手率暴增（239.6），交易成本吞噬收益

### 7.3 为什么 V4 (Long-Only) 胜出？

1. **不做空 = 不与结构性力量对抗**：黄金长期上行偏置强烈
2. **V3 的趋势主导正确识别了 Regime**：2011 做多、2013 做空、2019 做多
3. **Long-Only 约束消除了做空端的持续亏损**
4. **换手率最低（116.5）**：减少交易摩擦

### 7.4 V4 的局限

- 总收益 +17.99% 远逊于 B&H +141.12%
- 在趋势模糊期（2014-2015、2017-2018）频繁进出被震荡市打脸
- Sharpe +0.15 仍偏低，未达到"稳健策略"标准
